Reference: VibeCheck API

All endpoints, request shapes, and usage examples for lightweight and robust scans.

Search Quickstart

Start a scan, then fetch findings and logs.

curl -X POST https://vibecheck.aedify.ai/v1/assessments \
  -H "content-type: application/json" \
  -d '{
    "mode": "lightweight",
    "repo_url": "https://github.com/vulnerable-apps/damn-vulnerable-MCP-server"
  }'

Health

GET /v1/health

Health Check

Returns API availability, version, active tunnel count, and agent readiness.

{
  "status": "healthy",
  "version": "1.0.0",
  "active_tunnels": 0,
  "agents_available": true
}

Assessments

POST /v1/assessments

Create Assessment

Creates and queues either a lightweight static scan or a robust runtime scan.

Mode: lightweight | robust
Lightweight input: repo_url or files[]
Robust input: target_url, agents[], depth
Status: 202 Accepted (async processing)
{
  "mode": "robust",
  "target_url": "https://demo.example.com",
  "agents": ["recon", "auth", "injection", "config"],
  "depth": "deep",
  "idempotency_key": "scan-2026-03-03-001"
}
GET /v1/assessments

List Assessments

Paginated list with filtering and sorting.

Query: page, per_page, mode, status, sort
curl "https://vibecheck.aedify.ai/v1/assessments?mode=robust&status=complete&sort=-created_at"
GET /v1/assessments/{assessment_id}

Get Assessment

Fetches one assessment object, status, counts, and links.

POST /v1/assessments/{assessment_id}/rerun

Rerun Assessment

Resets findings/logs and queues scan again. Only allowed when status is complete or failed.

DELETE /v1/assessments/{assessment_id}

Delete Assessment

Deletes assessment and related findings/logs.

WS /v1/assessments/{assessment_id}/ws

Assessment Status Stream

WebSocket stream for real-time status updates and terminal completion event.

{
  "type": "assessment_update",
  "data": {
    "id": "asm_xxx",
    "status": "scanning",
    "finding_counts": { "total": 4 }
  }
}

Findings

GET /v1/assessments/{assessment_id}/findings

List Findings

Paginated findings for one assessment, filterable by severity, category, agent.

Query: page, per_page, severity, category, agent, sort
GET /v1/assessments/{assessment_id}/findings/{finding_id}

Get Finding

Returns one finding with severity, evidence, location, and remediation.

POST /v1/assessments/{assessment_id}/findings/{finding_id}/analyze

Analyze Finding

AI-enriched explanation for one finding: summary, impact, root cause, and action plan.

Agent Logs

GET /v1/assessments/{assessment_id}/logs

List Agent Logs

Returns robust mode reasoning/step logs. Lightweight mode returns logs_not_available.

Query: page, per_page, agent

Agents

GET /v1/agents

List Agents

Returns available agents and supported categories for robust/lightweight modes.

GET /v1/agents/{agent_name}

Get Agent

Returns one agent definition by name.

Memory

GET /v1/memory/status

Memory Status

Returns whether Supermemory integration is enabled.

GET /v1/memory/search

Search Memory

Searches historical findings semantically with optional assessment scoping.

Query: q, limit, assessment_id

Tunnel

WS /v1/tunnel

Tunnel WebSocket

Used by local client (`vibecheck connect`) to expose local app port for robust mode.

{
  "type": "connect",
  "target_port": 3000
}
GET /v1/tunnel/sessions

List Tunnel Sessions

Returns active and historical tunnel sessions.

GET /v1/tunnel/sessions/{session_id}

Get Tunnel Session

Returns one tunnel session by ID.

Error Format

error envelope

Consistent Error Payload

All application errors follow this shape:

{
  "error": {
    "type": "validation_error",
    "message": "Mode must be 'lightweight' or 'robust'.",
    "code": "INVALID_MODE",
    "param": "mode"
  }
}
